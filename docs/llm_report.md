# Отчет об изучении больших языковых моделей (LLM)

В рамках прохождения учебной практики было проведено ознакомление с современными большими языковыми моделями (LLM), принципами их архитектуры, промпт-инжинирингом и возможностями интеграции ИИ в прикладные проекты.

---

## 1. Пройденные онлайн-курсы
В процессе обучения были освоены следующие учебные курсы:
1.  **«Введение в архитектуру Transformer и LLM» (Stepik)**
    *   *Полученные знания:* Изучены теоретические основы механизма внимания (Self-Attention), архитектура энкодер-декодер (Transformer), этапы предобучения (Pre-training) и тонкой настройки (Fine-tuning) языковых моделей.
2.  **«ChatGPT Prompt Engineering for Developers» (DeepLearning.AI)**
    *   *Полученные знания:* Освоены базовые и продвинутые техники составления промптов: Zero-shot/Few-shot prompting, Chain-of-Thought (цепочки рассуждений), использование системных ролей (System/User/Assistant), структурирование выходных данных в формате JSON.
3.  **«Hugging Face NLP Course» (Hugging Face)**
    *   *Полученные знания:* Практическая работа с библиотекой `transformers` на Python. Загрузка предобученных моделей, токенизация текстовых данных, использование пайплайнов (pipelines) для классификации текстов, извлечения сущностей и анализа тональности.

## 2. Ключевые выводы и полученные навыки
*   **Понимание работы трансформеров:** Понимание того, как механизм внимания позволяет модели учитывать контекст всего предложения целиком (в отличие от рекуррентных сетей RNN/LSTM), что кардинально повышает качество генерации и анализа текстов.
*   **Промпт-инжиниринг как инструмент оптимизации:** Разработка эффективных шаблонов запросов позволяет решать сложные аналитические задачи без дорогостоящей дообучения моделей на локальном оборудовании.
*   **Этические и юридические аспекты ИИ:** Ознакомление с проблемами "галлюцинаций" моделей, предвзятости данных (data bias), конфиденциальности персональной информации студентов и правилами безопасного использования коммерческих API.

## 3. Применение LLM при выполнении практики
Большие языковые модели активно применялись как вспомогательный инструмент на всех этапах проектирования социологической системы:

### 3.1. Генерация синтетического датасета отзывов
Для проверки аналитического дашборда требовалось заполнить базу данных репрезентативными студенческими анкетами. С помощью LLM-ассистента была сгенерирована база из 1500 текстовых отзывов на русском языке по различным критериям оценки преподавателей:
*   *Промпт:* `Сгенерируй 10 реалистичных отзывов студентов о преподавателе вуза на русском языке. 4 отзыва должны быть позитивными (похвала за лекции, интерактив), 3 нейтральными (материал хороший, но скучно), 3 негативными (предвзятое отношение, плохая организация). Формат вывода: JSON-список объектов с полями "text" и "rating".`
*   *Результат:* Получен качественный набор тестовых текстовых отзывов, содержащих характерный студенческий сленг и профессиональные термины.

### 3.2. Помощь в разработке HTTP-сервера и регулярных выражений
*   При написании HTTP-сервера на сокетах с помощью ИИ были сгенерированы шаблоны регулярных выражений для парсинга сложных заголовков (например, заголовка `Content-Type: multipart/form-data; boundary=...` для извлечения границ файлов).
*   LLM помогла быстро составить обширный словарь сопоставления расширений файлов и MIME-типов (более 100 расширений), что сэкономило время ручного поиска спецификаций.

### 3.3. Разработка аналитических скриптов
Для анализа текстовых отзывов студентов на Python с использованием библиотеки `transformers` с помощью LLM был написан скрипт классификации отзывов на три класса тональности (позитивный, нейтральный, негативный) на основе русскоязычной модели `rubert-base-cased-sentiment`. Это позволило реализовать автоматический подсчет индекса удовлетворенности студентов без ручной разметки комментариев.
